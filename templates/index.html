Discovery Brief Agent

Trigger: HubSpot webhook (meeting booked)  ·  Model: Claude Opus 4.6  ·  Repo: discovery-brief-agent

Automatically researches a prospect before a discovery call. When a new meeting is booked in HubSpot, this agent pulls deal and contact data, enriches it via Amplemarket, runs live web research, and generates a 16-section MEDDPICC-style brief saved directly as a HubSpot note on the deal. It also advances the deal stage to "First Meeting Scheduled" so the pipeline stays accurate. The goal is for the AE to walk into every call knowing the prospect's integration landscape, pain points, and ICP fit — without doing any manual research.

Rules
🚦 Qualification gate
Only runs for deals in the Global Embedded Pipeline at the "Lead or Opportunity (Meeting Booked)" stage. Any other pipeline or stage → agent exits early with no brief generated. This is the most common reason a brief silently doesn't appear.
🔍 Signal Research reuse
Before running web research for §9 (Trigger Signals), the agent checks whether a Signal Research Agent note already exists on the deal. If found, it reuses that content directly — skipping the web search step entirely. If no signal note exists, it runs run_signal_research() to generate one fresh.
📊 Deal stage advancement
After every successful brief, the deal stage is automatically advanced to "First Meeting Scheduled" (stage ID: 113001644). This cannot be disabled on the auto-trigger flow.

Follow-up Tracker Agent

Trigger: Daily cron 18:00 UTC  ·  Model: Claude Sonnet 4.6  ·  Repo: followup-tracker-agent

Every evening, checks whether the sales team's HubSpot follow-up tasks for the day were actually executed. For each task due today, it gathers evidence — emails sent, calls logged, notes added — and asks Claude to classify the outcome as Completed, Incomplete, Missed, or Not enough data. Posts a structured Slack report to the appropriate channel based on deal stage, giving managers a clear daily view of follow-up discipline across the pipeline.

Rules
✅ Completed
Sufficient evidence the intended action was fully executed. "Follow up with X" → any email, call, or note logged today. "Book a meeting" → meeting scheduled or logged. "Send proposal" → email logged or note confirms sent.
⚠️ Incomplete
Some activity exists but doesn't fully meet the task intent — e.g. a note was added but no actual outreach was sent.
❌ Missed
No engagement of any kind logged today on the deal, and no next task exists to indicate follow-up activity was rescheduled.
❓ Not enough data
Task title is too ambiguous to determine intent, deal association is missing, or the action likely happened outside HubSpot (e.g. WhatsApp, phone).
🚫 Excluded from evidence
Agent-generated notes are excluded: Buying Intent / Intent Levels (AI), Weekly Review (AI), AI meeting evaluation, and any note starting with "Auto-generated by Albato" (Signal Research, Discovery Brief, etc.).

Pipeline Agent

Trigger: Daily cron 16:00 UTC  ·  Model: Claude Sonnet 4.6  ·  Repo: pipeline-agent

Runs daily to keep HubSpot deal stages accurate and CRM data clean. Applies five pipeline rules automatically: advancing deal stages when meetings are completed, fixing incorrect meeting types, extracting Business Fit scores from Sybill call transcripts using Claude, and backfilling contact engagement associations so activity history stays linked to deals. Removes the need for reps to manually update stages after each call.

Rules
Rule 2 — Advance to "2nd Meeting Booked"
Deal has 2+ meetings and the latest meeting is in Scheduled status → advance stage to "2nd Meeting Booked".
Rule 3 — Mark follow-up completed
Deal has 2+ meetings and all meetings are in Completed status → set status field to "Follow-up meeting completed".
Rule 4 — Post-Meeting Qualification + Business Fit extraction
Deal has exactly 1 meeting in Completed status → advance stage to "Post-Meeting Qualification". Claude reads the Sybill call transcript from HubSpot notes and extracts the Business Fit score, writing it back to the deal property.
Rule 5 — Fix mislabeled meeting types
Deals with multiple meetings where all are labeled "Outreach - 1st meeting" → relabel subsequent meetings as "Embedded Negotiations" to keep the meeting type data accurate.
Rule 6 — Backfill contact engagements
For every deal, finds linked contacts and copies their recent engagement records (last 2 weeks) as deal associations — ensuring emails and calls from contacts appear on the deal timeline even if not logged directly against it.
